# Turing Test 

## Description

It allows a user to evaluate two short conversations and decide **which one was written by a human and which was generated by an AI system**.

The program randomly selects a real human conversation from a dataset and generates a simulated AI conversation. These are then displayed as **Conversation A** and **Conversation B**. The user must guess which conversation is human.

The system keeps track of how often the **AI successfully fools the evaluator**, similar to real Turing Test experiments.

---

# Concept

The **Turing Test**, proposed by **Alan Turing (1950)**, is used to determine whether a machine can imitate human conversation convincingly enough that a human evaluator cannot reliably distinguish it from another human.

In this program:

1. A human chat log is selected from a dataset.
2. A simulated AI chat is generated dynamically.
3. The conversations are randomly assigned to **Chat A** or **Chat B**.
4. The user guesses which conversation is human.
5. The system records statistics on AI success.

# AI Conversation Generation

The AI chat is generated using a **dynamic template-based generator** that mimics casual online conversation.
It uses multiple categories of phrases:

* conversation starters
* responses
* follow-up questions
* casual endings

Random combinations of these elements create **new chat sequences each round**, making the AI behavior less predictable.

Example AI structure:

```
Opener → Response → Follow-up → Casual Ending
```

Example generated AI conversation:

```
sup
watching youtube lol
what about you
haha yeah

# Running the Program

```
./turing
```

The program will repeatedly run evaluation rounds until the user chooses to exit.

---

# Statistics

The program tracks:

* **Total Evaluations** – number of tests performed
* **AI Fooling Rate** – percentage of times the user incorrectly identified the AI

Formula:

```
AI Fooling Rate = (AI Fooled / Total Tests) × 100
```
